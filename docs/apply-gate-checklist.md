# Apply Gate Checklist

적용 전, 아래 4개를 모두 확인한다.

1. 외부 영향이 있는가?
- 예: 메시지 발송, 공개 포스팅, 배포, 외부 API 상태 변경
- 있으면: 반드시 사전 승인 필요

2. 승인 필요한 변경인가?
- 설정 변경, 업데이트, 보안 정책 수정, 자동화 추가
- 필요하면: "적용할까요?" 트리거 후 진행

3. 롤백 가능한가?
- 원복 방법이 문서화되어 있는가?
- 백업 또는 이전 상태로 되돌리는 절차가 있는가?

4. 성공/실패 기준이 있는가?
- 성공 신호: 상태/로그/동작 확인 방법
- 실패 신호: 에러/부작용/성능 저하 기준
