# Backlog Triage (Quick Rule-of-3)

Use this when deciding whether a stale task should stay in backlog.

## Keep if at least one is true
- **User impact is clear:** it directly improves reliability, speed, or UX for an active workflow.
- **Time-to-complete is small:** can be finished in about 1 hour or less.
- **Dependency unblocker:** it unlocks or de-risks another active task.

## Remove or archive if all are true
- No clear owner or use-case.
- No movement for 30+ days.
- No measurable outcome can be named in one sentence.

## Review cadence
- Run once weekly during maintenance.
