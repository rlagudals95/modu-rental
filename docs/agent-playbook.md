# Agent Playbook (OpenClaw 운영용)

목적: 정보 수집부터 적용 결정까지를 재현 가능한 방식으로 운영한다.

## 1) 기본 운영 원칙
- 한 사이클 = 원자적 변경 1개
- 적용 전 반드시 승인 받기
- 외부 반영(공개/전송/배포)은 별도 승인 없이는 금지
- 모든 변경은 롤백 가능해야 함

## 2) 적용 후보 수집 소스
- OpenClaw 공식 문서/릴리즈 노트
- OpenClaw GitHub 이슈/PR/릴리즈
- ClawHub 스킬/커뮤니티 적용 사례
- 사용자가 제공한 영상/아티클/링크

## 3) 후보 필터 기준
- 현재 환경(맥북 + 현 설정)에서 실효성 있는가?
- 위험도 대비 기대효과가 큰가?
- 적용/검증/롤백이 명확한가?

## 4) 보고 형식 (항목별)
- 항목명
- 기대효과
- 리스크
- 소요시간
- 롤백 방법
- 추천도: 적용 / 보류 / 비추천

## 5) 승인 트리거
항상 마지막에 아래 질문으로 종료:
- "적용할까요? (1번 / 1+3번 / 전부 / 보류)"

승인 전에는 실제 적용 금지.

## 6) 적용 후 보고
- 실제 변경 내용
- 검증 결과
- 문제 발생 여부
- 롤백 여부(필요시)
- 다음 개선 후보 1개
